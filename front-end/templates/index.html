    Google App Engine

    the muscle

    GAE is a great choice for a quick hack. Powered by Google's infrastructure, it takes care of all hosting aspects, including the database: a proprietary technology called the App Engine datastore. It's a little different from a standard Relational Database, and is the biggest change from an otherwise "normal" Django app. It also comes with a great SDK that lets you deploy with the touch of a button.

    You don't need to have ever used GAE before to use this hackpack. Direct interaction with GAE will probably be limited to installing the SDK, signing up, and hitting "deploy" for the most part.

    Learn more from Google »

    Python 2.7

    the something

    GAE only supported Python 2.5 until recently, and the upgrade offers some decent advantages. 2.7 support is still limited -- bugs remain -- but the updated language is a plus along with the much-needed capability of Multithreading.

    Familiarity with Python is a must if you plan on using this hackpack. You don't have to be a dedicated Pythonista by any means -- after all, it's a very easy language to pick up -- but I'll be assuming you know Python throughout this tutorial.

    Learn (about) Python »

    Django

    the skeleton

    The most popular web framework for Python, Django is a pretty full-featured framework for dynamic websites. It's very well suppported and has a strong community of developers who regularly put out add-ons and the like. Most standard questions about Django, and a great many non-standard questions, have already been well-answered on StackOverflow.

    Familiarity with Django is a major plus, but not strictly required, to use this hackpack. If nobody in your group has ever used Django before, this may be very tough for you if you're building something particularly complicated. I'll do my best to help out through tutorials and explanations of the basics. For those with 8-16 hours to spend very wisely, I strongly recommend learning more at djangobook.com.
